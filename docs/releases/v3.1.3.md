# Tracegate 3.1.3

Tracegate 3.1.3 completes the host-based production rollout started in 3.1.1.
It restores account-scoped SS2022 lifecycle handling inside Xray, corrects the
Telemt/Prometheus listener conflict and makes the release archive install every
canonical host systemd unit required by the current runtime.

Data-plane container services intentionally track their upstream `latest` tag
and pull before every start. The deployment process records the image IDs that
actually passed production verification so an operator can diagnose or roll
back a conflicting upstream update without reintroducing version locks into the
public release contract.

All Telegram Proxy grants issued before this rollout were revoked and removed.
Users must request a new account-bound grant from the Connections menu.
